## 4.2.0 亮点

- Command Hub、侧边面板、密码库和局域网传输采用统一的深色实体界面。
- Active Actions（活动功能）同步网页、Command Hub 与侧边面板，可关闭、撤销或恢复持续工具。
- “提取图片/视频”支持持续采集普通媒体、HLS、DASH、YouTube 与 Bilibili 页面信息，并明确标记受限资源。
- 全局隐私开关支持自定义防护和内置去广告规则；YouTube 使用兼容跳过模式。
- 局域网协议 v2 支持端到端加密、二进制高速通道、接收确认、背压、完整文件 SHA-256 与大文件临时存储。
- 手机与电脑双向保留文件 MIME（媒体类型）和扩展名，可识别被错误标记为 TXT/BIN 的 MP4、MOV 等媒体。
- 密码库增加快捷窗口、自动锁定、强化密钥派生和加密备份。

## 安装

### 使用发行包

1. 下载 [Web-Omni-vNext-4.2.0.zip](release/Web-Omni-vNext-4.2.0.zip)。
2. 将 ZIP 解压到固定目录。
3. 在 Chrome 打开 `chrome://extensions/`；在 Edge 打开 `edge://extensions/`。
4. 开启“开发者模式”或“开发人员模式”。
5. 点击“加载已解压的扩展程序”或“加载解压缩的扩展”。
6. 选择包含 `manifest.json` 的解压目录。
7. 在扩展菜单中固定 Web-Omni 图标，并刷新已打开的网页。

发行包同时包含 `native-host/`。它是可选的 Windows x64 本地助手，用于让手机在同一 Wi-Fi 下直接访问电脑，无需 GitHub Pages。

### 使用源代码

下载仓库源码并解压，然后在扩展管理页选择项目根目录。根目录应同时包含 `manifest.json`、`background.js`、`content-scripts/` 和 `side-panel/`。

### 更新版本

替换本地文件后，在扩展管理页点击 Web-Omni 卡片上的“重新加载”，随后刷新目标网页。重新加载扩展会结束当前标签页中的临时活动状态。

## 快速开始

1. 打开普通的 `http://` 或 `https://` 网页。
2. 点击工具栏中的 Web-Omni 图标，进入侧边面板。
3. 使用 `Ctrl+Shift+K` 打开 Command Hub，macOS 使用 `Command+Shift+K`。
4. 输入关键词搜索命令，或者进入“全部工具”按分类浏览。
5. 点击星标收藏常用命令；侧边面板会同步收藏，并支持拖动排序。
6. 持续运行的工具会出现在“当前活动”和网页侧边控制按钮中，可随时关闭或恢复。

### 快捷键

| Windows / Linux | macOS | 功能 |
| --- | --- | --- |
| `Ctrl+Shift+K` | `Command+Shift+K` | 打开或关闭 Command Hub |
| `Alt+S` | `Alt+S` | 清理当前页面的悬浮元素 |

浏览器可能占用部分组合键。可在 `chrome://extensions/shortcuts` 或 `edge://extensions/shortcuts` 中重新设置。

## 常用工作流

### 元素消除与恢复

1. 在 Command Hub 中运行“元素消除”。
2. 在页面上选择需要隐藏的元素。
3. 从“当前活动”使用“撤销”恢复最近一次操作。
4. 使用“全部恢复”清除当前页面的隐藏规则，或者进入“规则管理”查看记录。

元素通过隐藏标记处理，原始 DOM（文档对象模型）节点会保留，便于撤销和恢复。

### 密码库

1. 打开“密码金库”，设置至少 12 位主密码和 4–8 位数字安全码。
2. 解锁后添加站点、用户名、密码和备注。
3. 在登录页使用密码库快捷窗口执行一键填充。
4. 开启“自动保存”后，提交登录表单时会出现保存确认。
5. 在完整密码库中导出 v3 加密备份。

密码库闲置 5 分钟后自动锁定。从完整密码库复制凭据后，扩展会在约 30 秒后检查剪贴板，内容仍匹配时尝试清除；浏览器焦点和剪贴板权限可能影响清理结果。

### 局域网传输

1. 运行“局域网传输”，选择“自动”“在线”或“本地”。
2. 在线模式首次填写已部署的 HTTPS 手机页面地址；本地模式需要安装 Windows 本地助手。
3. 电脑生成带有临时会话和配对密钥的二维码，手机扫码后自动连接。
4. 连接后可拖入文件、选择文件、粘贴截图，或发送文本和链接。

协议 v2 使用端到端加密，并包含自适应分片、二进制高速通道、接收确认窗口、缺片重传、发送背压和完整文件 SHA-256 校验。较大文件使用 OPFS（浏览器私有文件系统）或 IndexedDB（浏览器本地数据库）暂存，电脑端确认浏览器下载完成后才清理临时文件。

#### Windows 本地助手

1. 在 `chrome://extensions` 或 `edge://extensions` 开启开发者模式，复制 Web-Omni 的 32 位扩展 ID。
2. 使用管理员 PowerShell 进入解压目录中的 `native-host/`。
3. 执行以下命令，将示例 ID 替换为实际 ID：

```powershell
.\install.ps1 -ExtensionId aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa -PrivateNetworkAccess Allow
```

4. 重新加载扩展，在局域网传输页选择“本地”或“自动”。手机与电脑需连接同一 Wi-Fi，Windows 网络配置应为“专用网络”。

本地助手绑定私有网络地址和随机端口，仅在局域网传输页使用期间运行，空闲后自动退出。中国大陆网络下建议优先使用本地模式，避免 GitHub Pages 和海外信令服务的可达性波动。

### 翻译与天气

- 侧边面板“翻译”支持选中文本、短句和整页翻译，整页翻译可恢复原文。
- “天气”支持城市搜索和浏览器定位，并显示当前天气与预报。
- 翻译会把文本片段和语言参数发送到 MyMemory。
- 天气会把城市名或定位坐标发送到 Open-Meteo。

### 屏幕录制

1. 运行“屏幕录制”。
2. 由浏览器选择要共享的标签页、窗口或屏幕。
3. 根据需要启用系统音频或麦克风。
4. 开始录制并在完成后保存文件。

支持 File System Access API（文件系统访问接口）时优先流式写入；内存降级模式达到 `512 MiB` 后会自动停止并保存。

## 数据与安全

### 密码库保护

- 凭据使用 AES-256-GCM 加密后保存在 `chrome.storage.local`。
- 密钥通过 PBKDF2-SHA-256、随机盐和 90 万次迭代派生。
- 主密码与安全码参与密钥派生；持久存储内容限于派生参数与密文。
- 解锁密钥仅保留在密码库页面内存中，锁定后清理运行状态。
- 旧版 60 万次迭代数据在成功解锁后换盐迁移。
- 导入限制为 5 MiB、最多 5000 条，并校验备份和密文结构。

浏览器密码库的安全性还取决于设备安全、浏览器配置和其他扩展。重要凭据建议保留独立的加密备份。

### 外部通信

| 功能 | 外部服务 | 发送内容 |
| --- | --- | --- |
| 翻译 | MyMemory | 待翻译文本片段、源语言和目标语言 |
| 天气 | Open-Meteo | 城市搜索词，或用户触发定位后得到的经纬度 |
| 局域网传输（在线） | 配置的手机页面、PeerJS 信令、Google/Cloudflare STUN | 页面主机可见常规访问日志；信令服务可见 Peer ID、SDP/ICE；文件与消息保持端到端密文 |
| 局域网传输（本地） | 无外部服务 | 手机页面、配对和加密中继均在当前私有网络内完成 |
| 媒体下载 | 页面资源源站 | 用户选择下载的资源请求 |

项目自身当前未集成独立遥测、广告或用户账户系统。使用翻译、天气、局域网传输和远程媒体功能时需要网络连接，并受相应服务可用性与隐私政策约束。

## 兼容性与限制

- 最低浏览器版本为 Chromium `114`。
- Chrome 内部页、Edge 内部页、扩展页、DevTools、`data:`、`file:` 和 `view-source:` 页面属于受限环境。
- YouTube 专用工具仅在 YouTube 页面启用。
- 页面标注和元素画中画需要有效的用户手势。
- 防截图模式只能遮挡网页内容，操作系统截图、外部拍摄和浏览器之外的录制仍可获取画面。
- Cookie 工具只能读取页面可访问的 Cookie；`HttpOnly` Cookie 由浏览器隔离，页面脚本无法读取。
- 追踪器、请求与 WebSocket 监听从启用时开始观察，已经发生的请求或已经加载的资源无法回溯。
- 在线局域网传输依赖 WebRTC、信令服务和当前网络策略；严格 NAT、防火墙或企业网络可能阻止连接。
- 本地模式要求手机与电脑位于可互访的同一私有网络；访客 Wi-Fi 的客户端隔离功能可能阻止连接。
- 媒体采集不会破解 YouTube 签名、合并受保护的 DASH 轨道或绕过 DRM（数字版权管理）。

## 常见问题

### Command Hub 无法打开

确认当前页面属于普通网站，随后在扩展管理页重新加载 Web-Omni，并刷新网页。浏览器内置页面和扩展商店页面禁止内容脚本注入。

### 密码库快捷窗口显示锁定

点击“更多”打开完整密码库，输入主密码和安全码完成解锁，再回到来源网页执行填充或保存。

### 局域网设备连接失败

在线模式下确认手机页面与 PeerJS 信令服务可访问，并重新生成会话。中国大陆网络或严格 NAT 环境优先使用 Windows 本地助手；同时检查两台设备处于同一 Wi-Fi、Windows 网络为“专用网络”，且防火墙允许本地助手访问私有网络。

### 视频保存成 TXT 或 BIN

4.2.0 会传递媒体类型并检查文件头，自动恢复 MP4、MOV、WebM、MKV、AVI 等扩展名。旧版本已经保存的文件通常只需根据原格式把后缀改回 `.mp4` 或 `.mov`，文件字节由完整性校验保证一致。

### 更新后网页仍显示旧界面

在扩展管理页点击“重新加载”，再刷新所有已打开的目标网页。旧标签页会继续保留旧内容脚本，刷新后使用新版本。

## 安装

### 使用发行包

1. 下载 [Web-Omni-vNext-4.2.0.zip](release/Web-Omni-vNext-4.2.0.zip)。
2. 将 ZIP 解压到固定目录。
3. 在 Chrome 打开 `chrome://extensions/`；在 Edge 打开 `edge://extensions/`。
4. 开启“开发者模式”或“开发人员模式”。
5. 点击“加载已解压的扩展程序”或“加载解压缩的扩展”。
6. 选择包含 `manifest.json` 的解压目录。
7. 在扩展菜单中固定 Web-Omni 图标，并刷新已打开的网页。

### 使用源代码

下载仓库源码并解压，然后在扩展管理页选择项目根目录。根目录应同时包含 `manifest.json`、`background.js`、`content-scripts/` 和 `side-panel/`。

### 更新版本

替换本地文件后，在扩展管理页点击 Web-Omni 卡片上的“重新加载”，随后刷新目标网页。重新加载扩展会结束当前标签页中的临时活动状态。

## 快速开始

1. 打开普通的 `http://` 或 `https://` 网页。
2. 点击工具栏中的 Web-Omni 图标，进入侧边面板。
3. 使用 `Ctrl+Shift+K` 打开 Command Hub，macOS 使用 `Command+Shift+K`。
4. 输入关键词搜索命令，或者进入“全部工具”按分类浏览。
5. 点击星标收藏常用命令；侧边面板会同步收藏，并支持拖动排序。
6. 持续运行的工具会出现在“当前活动”和网页侧边控制按钮中，可随时关闭或恢复。

### 快捷键

| Windows / Linux | macOS | 功能 |
| --- | --- | --- |

## 常用工作流

### 元素消除与恢复

1. 在 Command Hub 中运行“元素消除”。
2. 在页面上选择需要隐藏的元素。
3. 从“当前活动”使用“撤销”恢复最近一次操作。
4. 使用“全部恢复”清除当前页面的隐藏规则，或者进入“规则管理”查看记录。

元素通过隐藏标记处理，原始 DOM（文档对象模型）节点会保留，便于撤销和恢复。

### 密码库

1. 打开“密码金库”，设置至少 12 位主密码和 4–8 位数字安全码。
2. 解锁后添加站点、用户名、密码和备注。
3. 在登录页使用密码库快捷窗口执行一键填充。
4. 开启“自动保存”后，提交登录表单时会出现保存确认。
5. 在完整密码库中导出 v3 加密备份。

密码库闲置 5 分钟后自动锁定。从完整密码库复制凭据后，扩展会在约 30 秒后检查剪贴板，内容仍匹配时尝试清除；浏览器焦点和剪贴板权限可能影响清理结果。

### 局域网传输

1. 运行“局域网传输”，电脑端会生成 6 位房间码和二维码。
2. 下载手机页面，或导出局域网页包并通过本地网页服务打开。
3. 手机端输入房间码连接电脑。
4. 连接后可拖入文件、选择文件、粘贴截图，或发送文本和链接。

传输协议包含分片校验、批量确认、缺片重传、断线重试和发送背压控制。文件与消息通过 WebRTC DataChannel（数据通道）传输；PeerJS 信令服务参与连接协商。

### 翻译与天气

- 侧边面板“翻译”支持选中文本、短句和整页翻译，整页翻译可恢复原文。
- “天气”支持城市搜索和浏览器定位，并显示当前天气与预报。
- 翻译会把文本片段和语言参数发送到 MyMemory。
- 天气会把城市名或定位坐标发送到 Open-Meteo。

### 屏幕录制

1. 运行“屏幕录制”。
2. 由浏览器选择要共享的标签页、窗口或屏幕。
3. 根据需要启用系统音频或麦克风。
4. 开始录制并在完成后保存文件。

支持 File System Access API（文件系统访问接口）时优先流式写入；内存降级模式达到 `512 MiB` 后会自动停止并保存。

## 数据与安全

### 密码库保护

- 凭据使用 AES-256-GCM 加密后保存在 `chrome.storage.local`。
- 密钥通过 PBKDF2-SHA-256、随机盐和 90 万次迭代派生。
- 主密码与安全码参与密钥派生；持久存储内容限于派生参数与密文。
- 解锁密钥仅保留在密码库页面内存中，锁定后清理运行状态。
- 旧版 60 万次迭代数据在成功解锁后换盐迁移。
- 导入限制为 5 MiB、最多 5000 条，并校验备份和密文结构。

浏览器密码库的安全性还取决于设备安全、浏览器配置和其他扩展。重要凭据建议保留独立的加密备份。

### 外部通信

| 功能 | 外部服务 | 发送内容 |
| --- | --- | --- |
| 翻译 | MyMemory | 待翻译文本片段、源语言和目标语言 |
| 天气 | Open-Meteo | 城市搜索词，或用户触发定位后得到的经纬度 |
| 局域网传输 | PeerJS 信令、Google/Cloudflare STUN | Peer ID、SDP/ICE 连接协商信息；文件内容走 WebRTC 数据通道 |
| 媒体下载 | 页面资源源站 | 用户选择下载的资源请求 |

项目自身当前未集成独立遥测、广告或用户账户系统。使用翻译、天气、局域网传输和远程媒体功能时需要网络连接，并受相应服务可用性与隐私政策约束。

## 兼容性与限制

- 最低浏览器版本为 Chromium `114`。
- Chrome 内部页、Edge 内部页、扩展页、DevTools、`data:`、`file:` 和 `view-source:` 页面属于受限环境。
- YouTube 专用工具仅在 YouTube 页面启用。
- 页面标注和元素画中画需要有效的用户手势。
- 防截图模式只能遮挡网页内容，操作系统截图、外部拍摄和浏览器之外的录制仍可获取画面。
- Cookie 工具只能读取页面可访问的 Cookie；`HttpOnly` Cookie 由浏览器隔离，页面脚本无法读取。
- 追踪器、请求与 WebSocket 监听从启用时开始观察，已经发生的请求或已经加载的资源无法回溯。
- 局域网传输依赖 WebRTC、信令服务和当前网络策略；严格 NAT、防火墙或企业网络可能阻止连接。

## 常见问题

### Command Hub 无法打开

确认当前页面属于普通网站，随后在扩展管理页重新加载 Web-Omni，并刷新网页。浏览器内置页面和扩展商店页面禁止内容脚本注入。

### 密码库快捷窗口显示锁定

点击“更多”打开完整密码库，输入主密码和安全码完成解锁，再回到来源网页执行填充或保存。

### 局域网设备连接失败

优先让两台设备连接同一网络，确认 WebRTC 和 PeerJS 信令服务可访问，并重新生成房间码。企业代理、防火墙和严格 NAT 环境可能需要调整网络策略。

### 更新后网页仍显示旧界面

在扩展管理页点击“重新加载”，再刷新所有已打开的目标网页。旧标签页会继续保留旧内容脚本，刷新后使用新版本。
